PROJECT PURPOSE
================================================================================
Curator AI is an intelligent system that ingests academic materials (PDFs of 
textbooks, lecture notes, and solutions), processes them using OCR and text 
extraction, stores embeddings in a vector database, and provides an API for 
querying and generating academic assignments based on the knowledge base.


DIRECTORY STRUCTURE & CONTENTS
================================================================================

CORE APPLICATION FILES (Root Level)
------------------------------------

main.py
  - FastAPI application (the main entry point)
  - Defines 3 main endpoints: /upload, /search, /generate-assignment
  - Handles file uploads to data/academic folder
  - Powers the search functionality using LibraryManager
  - Generates PDF assignments using SynthesisEngine
  - Run with: uvicorn main:app --reload

  - Configuration management

  - Centralizes all path configurations

check_config.py

requirements.txt
  - Currently empty (dependencies should be documented here)

test_ingest.py
  - Basic module import test


SOURCE CODE (src/ folder)
------------------------------------

src/ingestor.py - PDF Ingestion Service
  - extract_text_from_pdf(): Reads PDFs using pdfplumber, falls back to 
    pytesseract for OCR
  - chunk_text(): Splits extracted text into 512-character chunks with 
    64-character overlap
  - Creates LangChain Document objects with source metadata

src/retriever.py - Query & Retrieval Engine
  - LibraryManager class manages vector database interactions
  - Uses Chroma for vector storage (similarity search)
  - Queries the knowledge base with k=3 results
  - Routes queries through OllamaLLM for AI-curated responses
  - Supports two response modes: "notes" (brief) or "detailed" (comprehensive)
  - Run with: python -m src.retriever

src/llm_gateway.py - LLM & Embedding Interfaces
  - RobustOllamaEmbeddings: Handles batch embedding requests to Ollama service
    * Uses "nomic-embed-text" model
    * Implements embed_documents() and embed_query() methods
    * Batch size of 2 for stability
  - OllamaLLM: Wrapper for LLM generation requests
    * Uses "llama3" model
    * Calls http://localhost:11434/api/generate

WORKFLOW EXAMPLE
================================================================================

1. User uploads a PDF via /upload
   → Stored in data/academic

2. Ingestion extracts text with OCR fallback
   → Creates chunks with metadata

3. Chunks are embedded via Ollama
   → Stored in Chroma vector database

4. User queries via /search
   → Similarity search returns top 3 results
   → Results contextualized and sent to Llama3
   → AI-curated response returned

5. Optional: /generate-assignment
   → Creates formatted PDF output

API ENDPOINTS
================================================================================

GET /
  - Health check endpoint
  - Returns: {"status": "online"}

POST /upload
  - Upload documents for processing
  - Parameters: file (UploadFile)
  - Saves to: data/academic/{filename}
  - Returns: {"message": "Successfully uploaded {filename}."}

GET /search
  - Search the academic library
  - Parameters: 
    * question (str) - Search query
    * request_notes (bool) - Optional: return notes format if true
  - Returns: {"answer": str, "sources": []}

POST /generate-assignment
  - Generate an AI-curated PDF assignment
  - Parameters: question (str, Form)
  - Process:
    1. Searches library for relevant content
    2. Generates clean, professional PDF
  - Returns: PDF file (application/pdf)
